# AI Startup Analyzer

AI-powered startup idea analysis platform. Submit an idea, get a multi-section report with scores, market insights, MVP plans, and go-to-market strategies generated by specialized AI agents.

## What You Get

- Summary and scoring across market demand, competition, execution difficulty, and profit potential
- Idea validation, target audience, market research, and competitor analysis
- MVP planning, monetization, and go-to-market recommendations
- Risk assessment, roadmap, business model, vision and mission, brand identity, and budget estimates

## Architecture

- Frontend: Next.js 16
- Backend: NestJS + BullMQ workers
- Database: Neon Postgres + Prisma migrations
- Queue and cache: Redis
- AI providers: Google Gemini and OpenRouter

## Quick Start

This repo is configured for Neon.

- Neon org: `org-round-rain-85943923`
- Neon project: `tiny-mud-62618983`

### Prerequisites

- Node.js 20+
- pnpm 10+
- Redis 7+
- At least one AI API key

### 1. Install dependencies

```bash
pnpm install
```

### 2. Configure environment

```bash
cp .env.example .env
```

Set at minimum:

```env
DATABASE_URL="postgresql://...-pooler.../neondb?sslmode=require"
DIRECT_DATABASE_URL="postgresql://.../neondb?sslmode=require"
JWT_SECRET="change-me"
REDIS_HOST="localhost"
REDIS_PORT=6379
REDIS_PASSWORD=""
GEMINI_API_KEY=""
OPENROUTER_API_KEY=""
```

`DATABASE_URL` must be the pooled Neon connection string for the running app.

`DIRECT_DATABASE_URL` must be the direct Neon connection string for Prisma migrations.

### 3. Start Redis

Local development without Redis auth:

```bash
docker run --name ai-analyzer-redis -p 6379:6379 -d redis:7-alpine
```

### 4. Prepare the database

Generate the Prisma client:

```bash
pnpm db:generate
```

Apply the committed baseline migration to your local or shared development database:

```bash
pnpm db:migrate:deploy
```

For future schema changes in development, create a new migration with:

```bash
pnpm db:migrate:dev --name your-change
```

### 5. Run the app

```bash
pnpm dev
```

- Frontend: `http://localhost:3000`
- Backend API: `http://localhost:4000`

## Database Workflow

Use Prisma migrations as the only supported schema workflow.

- `pnpm db:generate`: regenerate Prisma Client
- `pnpm db:migrate:dev --name <change>`: create and apply a development migration
- `pnpm db:migrate:deploy`: apply committed migrations to a target database
- `pnpm db:studio`: open Prisma Studio

Do not use `prisma db push` for shared, staging, or production databases.

## Production Deployment

### Docker Compose

```bash
cp .env.example .env.prod
docker compose --env-file .env.prod -f docker-compose.prod.yml up -d
```

Production requires these variables in `.env.prod`:

- `DATABASE_URL`
- `DIRECT_DATABASE_URL`
- `REDIS_PASSWORD`
- `JWT_SECRET`
- `GEMINI_API_KEY` or `OPENROUTER_API_KEY`
- `FRONTEND_URL`

The backend container runs `prisma migrate deploy` before the API starts. If migrations fail, the container exits and the app does not serve traffic.

### Netlify

For frontend-only deployment:

- Build command: `pnpm --filter @ai-analyzer/frontend build`
- Publish directory: `apps/frontend/.next`
- Required env var: `NEXT_PUBLIC_API_URL`

## Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `DATABASE_URL` | Yes | Neon pooled connection string used by the running app |
| `DIRECT_DATABASE_URL` | Yes | Neon direct connection string used by Prisma migrations |
| `JWT_SECRET` | Yes | Strong random string for JWT signing |
| `REDIS_HOST` | Dev default | Redis hostname |
| `REDIS_PORT` | Dev default | Redis port |
| `REDIS_PASSWORD` | Required in prod | Redis password used by BullMQ and the production Redis container |
| `FRONTEND_URL` | Recommended | Frontend URL for CORS and redirects |
| `BACKEND_PORT` | Optional | Backend server port, default `4000` |
| `GOOGLE_CLIENT_ID` | Optional | Google OAuth client ID |
| `GOOGLE_CLIENT_SECRET` | Optional | Google OAuth client secret |
| `GOOGLE_CALLBACK_URL` | Optional | Google OAuth callback URL |
| `ENABLE_AI_GROUNDING` | Optional | Enable AI-based report grounding |
| `ENABLE_RULE_GROUNDING` | Optional | Enable rule-based report grounding |

## API Reference

### Authentication

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/auth/register` | Register with email and password |
| `POST` | `/auth/login` | Login |
| `POST` | `/auth/refresh` | Refresh access token |
| `POST` | `/auth/logout` | Invalidate refresh token |
| `GET` | `/auth/google` | Start Google OAuth |
| `GET` | `/auth/google/callback` | OAuth callback |
| `GET` | `/auth/me` | Get current user profile |

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/analysis` | Create a new analysis |
| `GET` | `/analysis` | Get user analyses |
| `GET` | `/analysis/:id` | Get one analysis |
| `GET` | `/analysis/:id/progress` | Poll progress |
| `DELETE` | `/analysis/:id` | Delete analysis |
| `POST` | `/analysis/:id/retry` | Retry failed analysis |
| `POST` | `/analysis/:id/regenerate/:section` | Regenerate a section |
| `POST` | `/analysis/chat` | Chat about an analysis |
| `GET` | `/analysis/csrf-token` | Get CSRF token |
| `GET` | `/analysis/me/plan` | Get plan usage |
| `GET` | `/health` | Health check |

## Troubleshooting

### Analysis stuck in pending

- Ensure Redis is running and reachable
- Check backend logs for BullMQ connection errors

### Prisma workflow

After schema changes:

```bash
pnpm db:generate
pnpm db:migrate:dev --name describe-your-change
```

For a fresh environment using committed migrations:

```bash
pnpm db:migrate:deploy
```

### Legacy SQLite data

No `dev.db` file was present in this workspace during the Neon migration hardening pass, so there was no local SQLite data to import.
